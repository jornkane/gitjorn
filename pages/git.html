Viktige Git-kommandoer:

  • For å laste opp til Github: git push orign master men det kan hende at ikke mappene stemmer overens, da er det git pull origin master, før jeg tar git push origin master
  • Log i Git bør gjøres med oneline, slik: git log --oneline hvis ikke kan den bli fryktelig lang og jeg måtte avslutte VsCode for å bruke Git i terminal igjen...
  • Når vi angrer noe vi har adda, eller får feilmelding/advarsel ved addding har vi to valg: git reset som tar vekk filene fra staging eller git rm som fjerner den fra både staging og sletter den fra mappa

Fikse problemer med SSH-sertifikat

Litt hjelp fra computer.com for å fikse problem med SSH-sertifikat:
It seems like you are encountering an SSL certificate issue when trying to push to your GitHub repository. This error typically occurs when there is a problem with the SSL certificate chain. To resolve this, you can try the following steps:
1. **Update Git Configuration**: You can set Git to not verify the SSL certificate by running the following command: ``` git config --global http.sslVerify false ```
2. **Add GitHub's SSL Certificate**: You can also try adding GitHub's SSL certificate to your trusted certificates. You can find GitHub's SSL certificate by visiting [https://github.com](https://github.com) and then save it to a file.
3. **Use SSH**: Another option is to use SSH instead of HTTPS for your GitHub repository. This can be done by changing the remote URL to the SSH URL. You can find the SSH URL in your GitHub repository.
Try these steps and see if they help you resolve the SSL certificate issue when pushing to GitHub. Let me know if you need further assistance!
