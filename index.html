Huskelapper for gitkommandoer vi bruker ofte også.
Vkitige Git-kommandoer:
For å laste opp til Github: git push orign master

men det kan hende at ikke mappene stemmer overens, da er det
git pull origin master, før jeg tar git push origin master

Log i Git bør gjøres med oneline, slik: git log --oneline
hvis ikke kan den bli fryktelig lang og jeg måtte avslutte VsCode for å bruke Git i terminal igjen...
Når vi angrer noe vi har adda, eller får feilmelding/advarsel ved addding
har vi to valg: git reset som tar vekk filene fra staging
eller git rm som fjerner den fra både staging og sletter den fra mappa
kan jeg angre etter å stage den med git add . ?
Ja det er bare å skrive git restore --staged index.html
