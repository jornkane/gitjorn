Her har jeg tenkt å skrive litt om Git og Github

Viktige Git-kommandoer:

  • For å laste opp til Github: git push orign master men det kan hende at ikke mappene stemmer overens, da er det git pull origin master, før jeg tar git push origin master
  • Log i Git bør gjøres med oneline, slik: git log --oneline hvis ikke kan den bli fryktelig lang og jeg måtte avslutte VsCode for å bruke Git i terminal igjen...
  • Når vi angrer noe vi har adda, eller får feilmelding/advarsel ved addding har vi to valg: git reset som tar vekk filene fra staging eller git rm som fjerner den fra både staging og sletter den fra mappa

Noen nyttige/unyttige tips for Git i VSCode (lista lager jeg med shortcuten li*5):

  • git diff navnpåfil.noe, som viser hva som er blitt forandret i filen.
  • Last ned vscode extension: GitLens som viser når du har commita ting (på linja du befinner deg)
  • Hold alt og bruk piltast for å flytte linjer opp og ned

Slette repository fra Github
